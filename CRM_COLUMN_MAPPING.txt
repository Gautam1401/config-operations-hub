================================================================================
BUSINESS LOGIC RULES
================================================================================

CONFIGURATION STATUS:
---------------------
- Standard      = Configuration - Status contains "Standard"
- Copy          = Configuration - Status contains "Copy"
- Implementation = Configuration - Status contains "Implementation"
- Not Configured = Configuration - Status is blank/NA

PRE GO LIVE STATUS:
-------------------
- Pass     = BOTH Domain Updated = "Yes" AND Set Up Check = "Yes"
- Partial  = ONE of them = "Yes", other = "No"
- Fail     = BOTH Domain Updated = "No" AND Set Up Check = "No"
- Pending  = Any blank/NA values

GO LIVE TESTING STATUS:
-----------------------
Tests: Sample ADF (40%), Inbound Email (12.5%), Outbound Mail (12.5%), Data Migration (35%)

- GTG (Good to Go)       = ALL 4 tests = "Yes"
- Go Live Blockers       = Sample ADF = "No" OR Data Migration = "No"
- Non-Blockers           = Inbound Email = "No" OR Outbound Mail = "No"
- Both Blocker & Non-Blocker = Can have BOTH simultaneously
- Fail                   = ALL tests failed
- Testing Completed      = Assigned To is NOT blank/NA
- Exclude                = Future go-live dates (Go Live > Today)

SCORE CALCULATION:
------------------
Start with 100%
- Sample ADF = "No" → Deduct 40%
- Inbound Email = "No" → Deduct 12.5%
- Outbound Mail = "No" → Deduct 12.5%
- Data Migration = "No" → Deduct 35%

ESCALATION BUCKETS (Days to Go Live):
--------------------------------------
- Escalated  = < 25 days
- Critical   = 25-45 days
- On Track   = > 45 days

EXCLUSIONS:
-----------
- "Not Configured" stores are OUT OF SCOPE for analytics
- NA/blank test values should be EXCLUDED from status calculations
- Future go-live dates should be EXCLUDED from Go Live Testing
